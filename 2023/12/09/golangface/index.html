垃圾回收

常见的垃圾回收算法有哪些？

  • 引用计数
    每个对象维护一个引用计数，被创建或被赋值给其他对象时引用计数自动加 1，对象被销毁则引用计数减 1，当引用计数为 0 时，对象就可以被回收
    • 优点：实现简单，垃圾对象可以很快被回收
    • 缺点：无法处理循环引用的情况，引用计数的维护和更新需要消耗一定的资源
  • 标记清除
    从根节点开始遍历所有对象，能到达的对象标记为活跃对象，未被标记的对象则为垃圾对象，最后统一回收所有的垃圾对象
    • 优点：可以处理循环引用的情况
    • 缺点：需要遍历所有的对象，效率较低，需要暂停整个程序的运行（STW：Stop The World）
  • 分代回收
    将所有的对象分为新生代和老年代，新生代中的对象存活时间较短，老年代中的对象存活时间较长，新老生代的垃圾回收策略不同
    • 优点：提高了垃圾回收的效率
    • 缺点：增加了垃圾回收的复杂度

Go 1.3 使用标记清除，一开始是知道垃圾清理完毕才恢复程序运行，后来将垃圾清理放在 STW 后面，与程序并发执行

Go 1.5 使用三色标记

三色标记法是什么？

  • 白色：暂无对象引用的潜在垃圾，可能被回收
  • 灰色：活跃对象，需要被扫描，存在指向白色对象的外部指针，垃圾收集器回去扫描这些对象的子对象
  • 黑色：活跃对象，已经被扫描，包括不存在引用外部指针的对象和根对象可及的对象

步骤：

  1. 将所有的对象标记为白色
  2. 将根对象标记为灰色，将第一次遍历到的节点标记为灰色
  3. 遍历灰色节点，将灰色节点标记为黑色，将灰色节点的子节点标记为灰色
  4. 重复步骤 3，直到灰色节点为空
  5. 将白色节点回收

但由于 CPU 的调度，可能出现

  1. 白色对象被黑色对象引用
  2. 灰色对象和可达的白色对象关系被打断

因此拓展出了两种办法

  1. 强三色不变式：不允许白色对象被黑色对象引用
  2. 弱三色不变式：黑色对象可以引用白色对象，但是白色对象必须是灰色对象的子节点或者其链路上的节点

于是引入了屏障机制：在程序执行时加一个判断，满足则回调

为保障栈的效率，屏障只在堆上运行，栈在 GC 后用 STW 的方式重新扫描

屏障为插入屏障和删除屏障

  • 插入屏障：用强三色不变式，当黑色对象引用白色对象时，将白色对象标记为灰色对象
  • 删除屏障：用弱三色不变式，若灰色引用的白色对象被删除，则将白色对象标记为灰色对象

缺点是精度低，一个对象即使被删除也可能活过这一轮在下一轮被回收

Go 1.8 使用三色标记加混合写屏障

混合写屏障是什么？

实现弱三色不变式

  1. GC 开始时将栈上的所有可达对象标记为黑色
  2. GC 时栈上新建的对象标记为黑色
  3. 被删除引用的对象标记为灰色
  4. 被添加引用的对象标记为灰色

GC的触发时间：

  1. 对内存分配达到控制器计算的堆阈值（初始大小 GOGC），之后要到达上一次 GC 的两倍才会触发 GC
  2. 一定时间间隔触发 GC，由 runtime.forcegcperiod 控制，默认两分钟

GPM 调度和 CSP 模型

CSP 模型是什么？

  • CSP 模型是一种并发编程模型，通过通信来共享内存，而不是通过共享内存来通信

GPM 调度是什么？

  • GPM 调度是 Go 语言运行时的调度模型，G 代表 Goroutine，P 代表 Processor，M 代表 OS Thread（Machine）

G：Goroutine，代表一个协程，协程是 Go 语言并发的基本单位，协程是由 Go 语言运行时调度的，协程的调度是非抢占式的，协程的调度由 Go 语言运行时负责，协程的调度是基于协程的主动让出（yield）和主动抢占（preempt）的

P：Processor，代表一个处理器，处理器是协程的执行者，即 goroutine 的上下文， 包含了一组 goroutine 队列，每个队列与一组优先级关联

M：OS Thread，代表一个操作系统线程，负责将 Processor 与操作系统线程进行绑定，实现真正的并发执行

GPM 调度的流程：

  1. 启动
    • 程序启动时，系统初始化一组 M 和 P，M 和 P 一一对应，P 会创建一个 goroutine 队列
  2. 创建 goroutine
    • 当一个 goroutine 被创建时，会被放入一个 P 的 goroutine 队列中
    • 如果某个 P 的本地队列已满，会将多余的 goroutine 放入全局队列中
  3. 运行 goroutine
    • M 会从全局队列中取出一个可运行的 goroutine，如果 M 关联的 P 非空，优先从 P 的本地队列中取出一个可运行的 goroutine，如果 P 的本地队列为空，M 可能从其他 P 的本地队列中偷取一些 goroutine
    • M 会将 goroutine 绑定到自己的 P 上，并执行该 goroutine
  4. 阻塞和唤醒
    • 当 goroutine 阻塞时，P 会将其从运行队列移除
    • 阻塞的 goroutine 可能会释放关联的 P（P 满了）
